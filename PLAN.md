## Phase 2 — Core Infrastructure: Logger & Main Lifecycle

**Goal:** Implement the `Logger` module and the full cBot lifecycle (`on_start`, `on_bar_closed`, `on_tick`, `on_stop`, `on_error`) in `main.py` so the bot can start, run, and stop cleanly.

### 2.1 Logger (`core/logger.py`)

The `Logger` class wraps all output. It must be the **first module initialized** in `on_start`.

**Constructor:**
```python
def __init__(self, api, log_level: LogLevel, file_logging: bool, label_prefix: str)
```

**Methods to implement:**

| Method | Behavior |
|--------|----------|
| `debug(msg)` | Writes if log_level <= DEBUG |
| `info(msg)` | Writes if log_level <= INFO |
| `warning(msg)` | Always prefixes `[WARN]` |
| `error(msg, exc=None)` | Always prefixes `[ERROR]`; formats exception traceback |
| `trade_entry(instruction, result)` | Structured log: symbol, direction, volume, entry_price, SL, TP, strategy |
| `trade_exit(position, reason)` | Structured log: symbol, exit_price, pnl, duration, reason |
| `risk_action(msg)` | Prefixes `[RISK]`; always INFO level or above |
| `daily_summary(snapshot)` | Formatted daily P/L report |

**Implementation rules:**
- All output goes through `api.Print(formatted_string)`.
- Format: `[YYYY-MM-DD HH:MM:SS UTC] [LEVEL] [MODULE] message`
- If `file_logging=True` and running locally: write same lines to a `.log` file in the cTrader local data directory using Python `open()`. Wrap in try/except to handle Cloud restriction gracefully.
- File log: daily rotation — new file each day named `ArgoAlgo_YYYY-MM-DD.log`.
- Never raise exceptions; catch all I/O errors internally and fall back to `api.Print`.

### 2.2 TradingBot Lifecycle (`main.py`)

**`on_start(self)`** — sequence of operations:
1. Initialize `Logger` with parameters.
2. Log bot start banner (version, account, balance, leverage).
3. Parse `TradedSymbols` parameter string into a list.
4. Initialize `DataProvider` with symbol list and API object.
5. Initialize `RiskManager` with parameters and Logger.
6. Initialize `OrderExecutor` with API, Logger.
7. Initialize `StrategyEngine` with enabled strategies and parameters.
8. Initialize `UIPanel`.
9. Subscribe to position and order events (see §2.3).
10. Record `initial_balance` and `daily_start_balance` in `LocalStorage`.
11. Log "Bot started successfully."

**`on_bar_closed(self)`** — main loop:
1. If `_is_halted`: return early (log halted state once per bar).
2. Call `RiskManager.check_drawdown_limits()` — if breached, set `_is_halted = True`, call `_halt_trading()`.
3. Call `_check_session_filter()` — if outside hours, return.
4. Call `_check_spread_filter()` for each symbol — skip symbols with wide spread.
5. Call `DataProvider.update()`.
6. Call `StrategyEngine.evaluate()` → list of `TradeSignal`.
7. For each signal: call `RiskManager.validate(signal)` → `TradeInstruction`.
8. For each valid instruction: call `OrderExecutor.execute(instruction)`.
9. Call `UIPanel.update(snapshot)`.

**`on_tick(self)`** — tick handler (lightweight):
1. If trailing stop enabled: call `RiskManager.update_trailing_stops()`.
2. Update `UIPanel` equity display.

**`on_stop(self)`** — cleanup:
1. Log "Bot stopping."
2. Send daily summary notification.
3. If `FridayCloseEnabled` and it is Friday and past `FridayCloseHourUTC`: close all positions.
4. Log final balance and P/L.

**`on_error(self, error)`** — error handler:
1. Log error with full context.
2. Send push notification if error is critical.
3. Do not re-raise (let cTrader handle).

### 2.3 Event Subscriptions

Subscribe in `on_start` to:

```python
api.Positions.Opened  += self._on_position_opened
api.Positions.Modified += self._on_position_modified
api.Positions.Closed  += self._on_position_closed
api.PendingOrders.Created   += self._on_pending_order_created
api.PendingOrders.Filled    += self._on_pending_order_filled
api.PendingOrders.Cancelled += self._on_pending_order_cancelled
```

Each handler: log the event with full details (symbol, type, price, SL, TP, P/L if applicable).

### 2.4 Session & Day-of-Week Filters

Implement `_check_session_filter()` in `main.py`:
- Get current UTC time from `api.Server.Time`.
- Check if current hour is between `TradingStartHourUTC` and `TradingEndHourUTC`.
- Check if current weekday is in the parsed `TradeDaysOfWeek` list.
- Friday close: if today is Friday and current hour >= `FridayCloseHourUTC`, close all bot-managed positions and set `_is_halted = True` until next Monday.

### 2.5 Halt & Resume Logic

Maintain `_is_halted: bool` state on the `TradingBot` instance.

`_halt_trading(reason: str)`:
1. Set `_is_halted = True`.
2. Log `[RISK] Trading halted: {reason}`.
3. Send push notification.
4. Update UIPanel status to "HALTED".

Note: The bot does NOT auto-resume from a total drawdown halt — requires manual restart. Daily drawdown halt auto-resets at midnight (check `daily_start_balance` date on each `on_bar_closed`).

### 2.6 Phase 2 Acceptance Criteria

- [ ] Bot starts and logs the banner without errors in cTrader IDE.
- [ ] All lifecycle methods are invoked and produce log output.
- [ ] Session filter correctly skips `on_bar_closed` outside configured hours.
- [ ] `_halt_trading()` sets the flag and prevents any new order execution.
- [ ] Position/order event handlers log correct details.

## Phase 3 — Data Provider

**Goal:** Implement `DataProvider` to give strategies a clean, consistent interface to multi-symbol, multi-timeframe OHLCV data and indicators.

### 3.1 DataProvider Class (`core/data_provider.py`)

**Constructor:**
```python
def __init__(self, api, symbols: list[str], logger: Logger)
```

**Internal state:**
- `_symbols: dict[str, Symbol]` — cTrader Symbol objects, keyed by name.
- `_bars: dict[str, dict[TimeFrame, Bars]]` — OHLCV bars per symbol per timeframe.
- `_indicators: dict[str, dict]` — indicator instances per symbol.

**`initialize(timeframes: list[TimeFrame])`**:
1. For each symbol in `symbols`: retrieve `Symbol` object via `api.Symbol(name)`.
2. For each symbol and timeframe: retrieve `Bars` object via `api.GetBars(symbol, timeframe)`.
3. Initialize default indicator set for each symbol (see §3.2).
4. Log how many symbols and timeframes were loaded.

**`update()`**:
- Called on `on_bar_closed`. Refreshes any derived state if needed (most data is live via cTrader API references).

**`get_bars(symbol: str, timeframe: TimeFrame) -> Bars`**:
- Returns the `Bars` collection for the given symbol and timeframe.
- Raises `DataProviderError` if symbol/timeframe not loaded.

**`get_symbol(symbol: str) -> Symbol`**:
- Returns the cTrader `Symbol` object (contains pip value, min volume, etc.).

**`get_spread_pips(symbol: str) -> float`**:
- Returns `(ask - bid) / pip_size` for the symbol.

**`get_indicator(symbol: str, indicator_key: str) -> Indicator`**:
- Returns a pre-initialized indicator instance by key.

### 3.2 Indicator Initialization

Pre-initialize one instance of each indicator per symbol at `on_start` to avoid repeated construction costs:

| Key | Indicator | Parameters |
|-----|-----------|------------|
| `ema_fast` | EMA | `TF_FastEmaPeriod` |
| `ema_slow` | EMA | `TF_SlowEmaPeriod` |
| `adx` | ADX | `TF_AdxPeriod` |
| `atr` | ATR | `TF_AtrPeriod` (shared default 14) |
| `bollinger` | Bollinger Bands | `MR_BollingerPeriod`, `MR_BollingerDeviation` |
| `rsi` | RSI | `MR_RsiPeriod` |
| `adx_filter` | ADX | `MR_AdxFilterPeriod` |
| `donchian_high` | Highest High | `BO_DonchianPeriod` |
| `donchian_low` | Lowest Low | `BO_DonchianPeriod` |
| `atr_bo` | ATR | `BO_AtrPeriod` |

Use cTrader's `api.Indicators.ExponentialMovingAverage()`, `api.Indicators.AverageTrueRange()`, etc.

### 3.3 Multi-Timeframe Support

`DataProvider` loads bars for at minimum two timeframes per symbol:
- **Primary timeframe** (bar closed events) — passed in from `TradingBot`.
- **Higher timeframe** for trend context — `H4` by default; configurable parameter `HigherTimeFrame`.

Strategies access higher-timeframe data via `data_provider.get_bars(symbol, TimeFrame.Hour4)`.

### 3.4 Spread Filter Check

`DataProvider.is_spread_acceptable(symbol: str, max_spread_pips: float) -> bool`:
- Computes current spread in pips using symbol's `PipSize`.
- Returns `True` if spread <= `max_spread_pips`.

## Phase 4 — Risk Management Engine

**Goal:** Implement the `RiskManager` — the most critical module. Every trade must pass through it before execution.

### 4.1 RiskManager Class (`core/risk_manager.py`)

**Constructor:**
```python
def __init__(self, api, params: RiskParams, logger: Logger, data_provider: DataProvider)
```

Where `RiskParams` is a dataclass holding all risk-related parameters from the cBot parameter group.

**Internal state:**
```python
_initial_balance: float      # set at on_start
_daily_start_balance: float  # reset at midnight
_daily_start_date: date      # to detect day rollover
_high_water_mark: float      # tracks peak equity
_trailing_stop_map: dict[int, float]  # position_id -> activated_sl_price
```

### 4.2 Position Sizing

**`calculate_volume(symbol: str, stop_loss_pips: float) -> float`**:

Formula:
```
risk_amount = api.Account.Balance * (RiskPerTradePercent / 100)
pip_value   = data_provider.get_symbol(symbol).PipValue
volume      = risk_amount / (stop_loss_pips * pip_value)
```

Then normalize:
- `volume = max(symbol.VolumeInUnitsMin, min(volume, symbol.VolumeInUnitsMax))`
- Round to nearest `symbol.VolumeInUnitsStep`.

Log the calculation at DEBUG level.

### 4.3 Signal Validation

**`validate(signal: TradeSignal) -> TradeInstruction`**:

Run the following checks in order. Fail fast — return rejected `TradeInstruction` on first failure.

| Check | Condition to PASS | Rejection Reason |
|-------|-------------------|------------------|
| Direction check | `signal.direction != NONE` | "No signal direction" |
| Spread check | `data_provider.is_spread_acceptable(symbol, MaxSpreadPips)` | "Spread too wide: {n} pips" |
| Session check | Currently in trading hours | "Outside trading session" |
| Drawdown daily | Daily drawdown < MaxDailyDrawdownPercent | "Daily drawdown limit reached" |
| Drawdown total | Total drawdown < MaxTotalDrawdownPercent | "Total drawdown limit reached" |
| Max concurrent | Open positions count < MaxConcurrentPositions | "Max concurrent positions reached" |
| Max per symbol | Positions for this symbol < MaxPositionsPerSymbol | "Max positions for {symbol} reached" |
| Volume check | Calculated volume >= symbol.VolumeInUnitsMin | "Calculated volume below minimum" |

If all checks pass: return `TradeInstruction(signal, volume, validated=True)`.
If any fails: return `TradeInstruction(signal, 0, validated=False, rejection_reason=...)` and log at INFO.

### 4.4 Drawdown Monitoring

**`check_drawdown_limits() -> DrawdownStatus`**:

Returns enum: `OK | DAILY_LIMIT_BREACHED | TOTAL_LIMIT_BREACHED`.

```python
current_equity = api.Account.Equity

# Daily drawdown
daily_loss_pct = (daily_start_balance - current_equity) / daily_start_balance * 100
if daily_loss_pct >= MaxDailyDrawdownPercent:
    return DrawdownStatus.DAILY_LIMIT_BREACHED

# Total drawdown from high-water mark
high_water_mark = max(high_water_mark, current_equity)
total_drawdown_pct = (high_water_mark - current_equity) / high_water_mark * 100
if total_drawdown_pct >= MaxTotalDrawdownPercent:
    return DrawdownStatus.TOTAL_LIMIT_BREACHED

return DrawdownStatus.OK
```

**Day rollover detection** (call at start of each `on_bar_closed`):
```python
today = api.Server.Time.date()
if today != _daily_start_date:
    _daily_start_balance = api.Account.Balance
    _daily_start_date = today
    logger.info("Daily balance reset: {balance}")
```

### 4.5 Stop-Loss Calculation

**`calculate_sl_pips(symbol: str, direction: Direction, sl_type: SLType, atr_multiplier: float) -> float`**:

- `SLType.ATR`: `atr_value = data_provider.get_indicator(symbol, 'atr').Result.Last(0)` → `sl_pips = (atr_value / pip_size) * atr_multiplier`
- `SLType.FIXED`: return the fixed pip value from strategy parameters.
- Enforce minimum SL: never less than `symbol.MinStopLossInPips` (from cTrader API).

### 4.6 Take-Profit Calculation

**`calculate_tp_pips(sl_pips: float, rr_ratio: float) -> float`**:
- `tp_pips = sl_pips * rr_ratio`

For mean reversion (target middle band):
- `calculate_tp_to_price(entry_price, target_price, direction, pip_size) -> float`

### 4.7 Trailing Stop Management

**`update_trailing_stops()`** — called on every `on_tick`:

For each open position managed by the bot (identified by label prefix):
1. Get current unrealized profit in pips.
2. If profit >= `TrailingStopTriggerPips` and trailing not yet active for this position:
   - Move SL to breakeven (`position.ModifyStopLossPrice(entry_price)`).
   - Mark as active in `_trailing_stop_map`.
3. If trailing already active:
   - Compute new SL = current_price - `TrailingStopDistancePips * pip_size` (for longs).
   - If new SL > current SL: update `position.ModifyStopLossPrice(new_sl)`.

Performance note: avoid calling `ModifyStopLossPrice` if SL hasn't moved by at least 1 pip (reduces API rate usage).

### 4.8 Phase 4 Acceptance Criteria

- [ ] `calculate_volume()` returns a value within symbol min/max and correctly rounded.
- [ ] `validate()` rejects a signal when spread exceeds max.
- [ ] `validate()` rejects when max concurrent positions reached.
- [ ] `check_drawdown_limits()` returns `DAILY_LIMIT_BREACHED` when daily loss equals threshold.
- [ ] `check_drawdown_limits()` returns `TOTAL_LIMIT_BREACHED` when equity drops by threshold%.
- [ ] Daily balance resets correctly at UTC midnight.
- [ ] Trailing stop moves to breakeven after `TrailingStopTriggerPips` profit.
- [ ] Trailing stop advances but never moves backward.

## Phase 5 — Strategy Implementation

**Goal:** Implement the three trading strategies plus the strategy selection/orchestration logic.

### 5.1 Base Strategy Interface (`strategies/base_strategy.py`)

```python
from abc import ABC, abstractmethod

class IStrategy(ABC):
    name: str  # class-level constant

    def __init__(self, api, data_provider: DataProvider, logger: Logger, params: dict): ...

    @abstractmethod
    def evaluate(self, symbol: str) -> TradeSignal:
        """Evaluate market conditions and return a TradeSignal."""

    @abstractmethod
    def should_close(self, position) -> bool:
        """Return True if this strategy wants to close the given position."""

    def on_position_closed(self, position): ...  # optional hook for state cleanup
```

All strategy classes inherit from `IStrategy`. Each `evaluate()` call is stateless — it reads from `DataProvider` and returns a signal.

### 5.2 Strategy 1: Trend Following (`strategies/trend_following.py`)

**Class:** `TrendFollowingStrategy(IStrategy)`
**name = "TrendFollowing"**

**`evaluate(symbol: str) -> TradeSignal`**:

```
bars       = data_provider.get_bars(symbol, primary_timeframe)
ema_fast   = data_provider.get_indicator(symbol, 'ema_fast')
ema_slow   = data_provider.get_indicator(symbol, 'ema_slow')
adx        = data_provider.get_indicator(symbol, 'adx')
atr        = data_provider.get_indicator(symbol, 'atr')

# Current and previous bar values
ema_fast_now  = ema_fast.Result.Last(0)
ema_fast_prev = ema_fast.Result.Last(1)
ema_slow_now  = ema_slow.Result.Last(0)
ema_slow_prev = ema_slow.Result.Last(1)
adx_value     = adx.ADX.Last(0)
close_price   = bars.ClosePrices.Last(0)
```

**BUY signal:**
- `ema_fast_prev <= ema_slow_prev` AND `ema_fast_now > ema_slow_now` (crossover up)
- `adx_value >= TF_AdxThreshold`
- `close_price > ema_slow_now`

**SELL signal:** Mirror conditions with crossover down.

**SL/TP:**
- `sl_pips = (atr.Result.Last(0) / pip_size) * TF_StopLossAtrMultiplier`
- `tp_pips = sl_pips * TF_TakeProfitRR`

**`should_close(position) -> bool`**:
- Return True if fast EMA crosses back in the opposite direction of the trade.

**Edge cases:**
- Skip if `adx_value` is NaN or bars have fewer than `max(TF_SlowEmaPeriod, TF_AdxPeriod) + 5` candles.
- Include `metadata` in signal: `{"adx": adx_value, "ema_fast": ema_fast_now, "ema_slow": ema_slow_now}`.

### 5.3 Strategy 2: Mean Reversion (`strategies/mean_reversion.py`)

**Class:** `MeanReversionStrategy(IStrategy)`
**name = "MeanReversion"**

**`evaluate(symbol: str) -> TradeSignal`**:

```
bollinger  = data_provider.get_indicator(symbol, 'bollinger')
rsi        = data_provider.get_indicator(symbol, 'rsi')
adx_filter = data_provider.get_indicator(symbol, 'adx_filter')
bars       = data_provider.get_bars(symbol, primary_timeframe)

upper_band  = bollinger.Top.Last(0)
lower_band  = bollinger.Bottom.Last(0)
middle_band = bollinger.Main.Last(0)
rsi_value   = rsi.Result.Last(0)
adx_value   = adx_filter.ADX.Last(0)
close_price = bars.ClosePrices.Last(0)
```

**BUY signal** (oversold bounce):
- `close_price < lower_band`
- `rsi_value < MR_RsiOversold`
- `adx_value < MR_AdxFilterThreshold` (market is ranging)

**SELL signal** (overbought rejection):
- `close_price > upper_band`
- `rsi_value > MR_RsiOverbought`
- `adx_value < MR_AdxFilterThreshold`

**SL/TP:**
- `sl_pips`: ATR-based (1.0x ATR as tighter stop for mean reversion)
- `tp_pips`: calculated to `middle_band` using `calculate_tp_to_price()`

**`should_close(position) -> bool`**:
- Return True if price reaches `middle_band` (target achieved).

**Edge cases:**
- Skip if any indicator returns NaN.
- Skip if already in a position for the symbol in the same direction.

### 5.4 Strategy 3: Breakout (`strategies/breakout.py`)

**Class:** `BreakoutStrategy(IStrategy)`
**name = "Breakout"**

**`evaluate(symbol: str) -> TradeSignal`**:

```
bars          = data_provider.get_bars(symbol, primary_timeframe)
donchian_high = data_provider.get_indicator(symbol, 'donchian_high')  # Highest(period)
donchian_low  = data_provider.get_indicator(symbol, 'donchian_low')   # Lowest(period)
atr_bo        = data_provider.get_indicator(symbol, 'atr_bo')

upper_channel = donchian_high.Result.Last(1)  # Previous bar's high (avoid lookahead)
lower_channel = donchian_low.Result.Last(1)
middle_channel = (upper_channel + lower_channel) / 2
atr_value     = atr_bo.Result.Last(0)
close_price   = bars.ClosePrices.Last(0)
```

**BUY signal** (upward breakout):
- `close_price > upper_channel`
- `atr_value / pip_size >= BO_AtrMinThreshold / pip_size` (sufficient volatility)
- No existing long position for this symbol

**SELL signal:** Mirror with `close_price < lower_channel`.

**SL/TP:**
- `sl_pips = (atr_value / pip_size) * BO_StopLossAtrMultiplier`
- `tp_pips = sl_pips * BO_TakeProfitRR`

**`should_close(position) -> bool`**:
- Return True if price retraces to `middle_channel`.

**Pending order variant (optional):**
- Instead of market order on close, place a stop-order at `upper_channel + 1 pip` at bar open. Cancel if not filled by bar close.

### 5.5 Strategy Engine (`core/strategy_engine.py`)

**`StrategyEngine`** orchestrates all active strategies.

**Constructor:**
```python
def __init__(self, strategies: list[IStrategy], mode: StrategyMode, adx_threshold: float, logger: Logger)
```

**`evaluate(symbols: list[str]) -> list[TradeSignal]`**:

```
signals = []
for symbol in symbols:
    active = _select_strategies(symbol)
    for strategy in active:
        signal = strategy.evaluate(symbol)
        if signal.direction != Direction.NONE:
            signals.append(signal)
            logger.debug(f"Signal from {strategy.name}: {signal}")
return signals
```

**`_select_strategies(symbol: str) -> list[IStrategy]`**:

- `StrategyMode.MANUAL`: return only enabled strategy instances.
- `StrategyMode.ADX_SWITCHING`:
  - Get current ADX value from `data_provider`.
  - If `adx > adx_threshold`: return `[TrendFollowingStrategy]`.
  - Else: return `[MeanReversionStrategy]`.
  - `BreakoutStrategy` can run independently alongside either mode if enabled.

**`check_exits(positions) -> list[tuple[position, str]]`**:
- For each open bot-managed position, ask the originating strategy `should_close(position)`.
- Return list of `(position, reason)` for positions that should close.
- Match position to strategy via the `label` field on the order (e.g., `"TF_EURUSD"`).

### 6.1 OrderExecutor Class (`core/order_executor.py`)

**Constructor:**
```python

```

**`execute(instruction: TradeInstruction) -> ExecutionResult`**:

1. Validate `instruction.validated == True`; log error and return failed result otherwise.
2. Build label: `f"{label_prefix}_{instruction.signal.strategy_name[:2].upper()}_{symbol}"`
3. Call `api.ExecuteMarketOrder(symbol, trade_type, volume, label, sl_pips, tp_pips)`.
4. Handle result:
   - On success: log trade entry via `Logger.trade_entry()`, return `ExecutionResult(success=True, position_id=...)`.
   - On error (e.g., insufficient margin, requote): log error, do NOT retry automatically (to avoid rate-limit issues). Return `ExecutionResult(success=False, error=...)`.

**`close_position(position, reason: str) -> bool`**:

1. Call `position.Close()`.
2. On success: log via `Logger.trade_exit()`.
3. Return `True` on success, `False` on error.

```python
closed_count = 0
for position in api.Positions:
    if position.Label.startswith(label_prefix):
        if self.close_position(position, reason):
            closed_count += 1
return closed_count
```

Used by the Panic Button and halt logic.

**`modify_sl(position, new_sl_price: float) -> bool`**:

**`place_pending_stop_order(symbol, direction, volume, stop_price, sl_pips, tp_pips, label, expiry_bars) -> PendingOrderResult`**:

- Before each call: check counter against limit (500 orders/min, 100 cancel/min, 1000 SL mod/min per PRD §3.4).

### 8.3 Custom Fitness Function

Create a separate file `optimization/fitness.py` implementing the custom fitness function for cTrader's optimizer:

```python
def calculate_fitness(backtesting) -> float:
    """
    Custom fitness: rewards profit, efficiency, and penalizes drawdown.
    Formula: Net Profit * Profit Factor / Max Drawdown
    Disqualifiers: fewer than 200 trades, max drawdown > 20%
    """
    net_profit   = backtesting.NetProfit
    profit_factor = backtesting.ProfitFactor
    max_drawdown  = backtesting.MaxEquityDrawdownPercentages

    if backtesting.TotalTrades < 200:
        return -1.0
    if max_drawdown > 20.0:
        return -1.0
    if net_profit <= 0:
        return -1.0

    return net_profit * profit_factor / max(max_drawdown, 0.01)
```

### 8.4 Optimization Process

**Step 1 — In-Sample Optimization (70% of data: 2021–2024)**

Use cTrader's optimizer with **Genetic Algorithm** mode on the following parameter ranges:

*Trend Following:*
```
FastEmaPeriod:         8–20 (step 2)
SlowEmaPeriod:        20–50 (step 5)
AdxThreshold:         20–35 (step 5)
StopLossAtrMultiplier: 1.5–3.0 (step 0.5)
TakeProfitRR:          1.5–3.5 (step 0.5)
```

*Mean Reversion:*
```
BollingerPeriod:      15–30 (step 5)
BollingerDeviation:    1.5–3.0 (step 0.5)
RsiOversold:          25–35 (step 5)
RsiOverbought:        65–75 (step 5)
AdxFilterThreshold:   20–30 (step 5)
```

*Breakout:*
```
DonchianPeriod:       10–30 (step 5)
AtrMinThreshold:     variable
StopLossAtrMultiplier: 1.0–2.5 (step 0.5)
TakeProfitRR:          1.5–3.5 (step 0.5)
```

Take the **top 10 parameter sets** by fitness score.

**Step 2 — Out-of-Sample Validation (30% of data: 2025)**

Run the top 10 parameter sets on unseen data. Accept only sets where:
- Out-of-sample performance >= 70% of in-sample performance (Profit Factor, Drawdown).
- Keep the 3 best sets per strategy.

**Step 3 — Walk-Forward Analysis**

Divide 5 years into 10 windows of 6 months:
- Each window: 4 months in-sample + 2 months out-of-sample.
- Slide forward by 2 months each iteration.
- Document efficiency ratio: `WFA_efficiency = avg(out-of-sample PF) / avg(in-sample PF)`.
- Accept strategy if efficiency >= 0.70.

**Step 4 — Sensitivity Analysis**

Take the best parameter set and perturb each parameter individually by ±10% and ±20%.
- If fitness drops > 30% with ±10% change: flag as potentially overfitted.
- Document results in a sensitivity table.

### 8.5 Backtest Documentation

For each strategy, record results in `tests/backtest_results_STRATEGY.md`:
- Date of backtest run
- Data range and source
- Final parameter set chosen
- All 8 KPI metrics
- Equity curve screenshot (attach)
- WFA efficiency ratio
- Sensitivity analysis summary
- Decision: PASS / FAIL / NEEDS_REVISION

- [ ] All three strategies pass the KPI minimums on out-of-sample data.
- [ ] WFA efficiency ratio >= 0.70 for all three strategies.
- [ ] Custom fitness function is accepted by cTrader optimizer without errors.

## Appendix B — Key Design Decisions

| Decision | Choice | Rationale |
|----------|--------|-----------|
| Language | Python | Native cTrader support, rich ecosystem, rapid dev |
| Deployment | cTrader Cloud | Free, 24/7, no VPS needed |
| Primary signal event | `on_bar_closed` | Avoids over-computation; most strategies are bar-based |
| Trailing stop event | `on_tick` | Requires tick-level responsiveness |
| Position sizing | Fixed fractional (1% risk) | Industry standard; limits single-trade damage |
| Stop-loss type | ATR-based (default) | Adapts to market volatility automatically |
| Strategy selection | Manual or ADX-switching | Gives control; ADX switching is evidence-based |
| Backtest data | Tick data (server) | Highest accuracy; simulates real spreads |
| Optimization target | Fitness = NP * PF / DD | Balances profit, consistency, and capital protection |
| Position labeling | `ArgoAlgo_{ST}_{SYM}` | Enables bot-managed position filtering |
